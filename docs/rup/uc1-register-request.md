## UC1. Зарегистрировать заявку на обслуживание

### 1. Идентификатор и название

- **Идентификатор**: UC1  
- **Название**: Зарегистрировать заявку на обслуживание

### 2. Краткое описание

Система обеспечивает регистрацию заявки на обслуживание физического объекта с автоматическим присвоением приоритета и определением применимого SLA.

### 3. Акторы

- Основной актор: Клиент / Система мониторинга.
- Второстепенные акторы: Диспетчер.

### 4. Предусловия

- Объект обслуживания (Asset) существует в Системе либо может быть создан в ходе сценария.
- Актор авторизован и обладает достаточными правами для регистрации заявки (для Клиента/Диспетчера).

### 5. Постусловия

- Создана и сохранена новая заявка (ServiceRequest) с уникальным идентификатором.
- К заявке привязан объект обслуживания.
- Заявке присвоен приоритет и определён применимый SLA.
- В журнале истории зафиксировано исходное изменение статуса заявки.

### 6. Основной поток событий

1. Актор инициирует создание новой заявки на обслуживание.
2. Система запрашивает данные об объекте обслуживания и описание проблемы.
3. Актор предоставляет данные об объекте (выбор из справочника или указание нового объекта) и описание проблемы.
4. Система при необходимости создаёт новый объект обслуживания и связывает его с заявкой.
5. Система определяет приоритет заявки на основе настроенных правил.
6. Система определяет применимый SLA для заявки (по договору, типу услуги, приоритету и другим параметрам).
7. Система создаёт запись заявки с присвоением идентификатора, приоритета, SLA и исходного статуса (например, «Новая»).
8. Система фиксирует изменение статуса заявки в журнале истории.
9. Система подтверждает успешное создание заявки актору.

### 7. Альтернативные потоки

- **AF1: Объект отсутствует в справочнике**
  - 3a. Актор выбирает опцию создания нового объекта обслуживания.
  - 3b. Система запрашивает обязательные атрибуты объекта.
  - 3c. Актор вводит данные об объекте.
  - 3d. Система создаёт объект и возвращается к шагу 5 основного потока.

- **AF2: Невозможно автоматически определить SLA**
  - 6a. Система не находит подходящий SLA по текущим настройкам.
  - 6b. Система присваивает заявке статус, указывающий на необходимость уточнения SLA, и уведомляет Диспетчера.
  - 6c. Заявка сохраняется, но помечается как требующая ручной обработки SLA.

### 8. Исключительные ситуации

- Ошибка сохранения данных заявки.
  - Система регистрирует ошибку, информирует актора о невозможности создания заявки и не сохраняет частичные данные.
