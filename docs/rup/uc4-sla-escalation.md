## UC4. Контролировать соблюдение SLA и управлять эскалациями

### 1. Идентификатор и название

- **Идентификатор**: UC4  
- **Название**: Контролировать соблюдение SLA и управлять эскалациями

### 2. Краткое описание

Система автоматически контролирует сроки выполнения заявок в соответствии с установленными SLA, выявляет риск и факт нарушения SLA и инициирует процедуры эскалации в соответствии с настроенными правилами.

### 3. Акторы

- Основной актор: Система (фоновый процесс/шедулер).
- Второстепенные акторы: Руководитель сервиса, Диспетчер.

### 4. Предусловия

- Для заявок определены приоритеты и связанные с ними SLA.
- В Системе настроены правила эскалаций.

### 5. Постусловия

- Заявки с риском нарушения SLA идентифицированы и помечены соответствующим образом.
- Для заявок, для которых условия эскалации выполнены, инициированы предусмотренные действия.
- Факты эскалаций и выполненные действия зафиксированы в журнале.

### 6. Основной поток событий

1. По расписанию Система инициирует процесс проверки активных заявок.
2. Система получает список активных заявок и соответствующие параметры SLA.
3. Для каждой заявки Система рассчитывает оставшееся время до контрольных точек SLA (реакция, решение).
4. Система сопоставляет результат с пороговыми значениями, заданными в правилах эскалации.
5. При достижении пороговых значений Система формирует запись о срабатывании эскалации.
6. Система выполняет предписанные правилами эскалации действия (например, уведомляет Руководителя сервиса и/или Диспетчера, повышает приоритет заявки, изменяет ответственного исполнителя).
7. Система фиксирует факт эскалации и выполненные действия в истории.
8. После обработки всех активных заявок процесс завершается до следующего запуска по расписанию.

### 7. Альтернативные потоки

- **AF1: Обновление правил эскалации**
  - 4a. При изменении правил эскалации Система должна использовать новые правила при следующем цикле проверки, без необходимости перезапуска сервисов.

### 8. Исключительные ситуации

- Недоступность данных SLA или правил эскалации.
  - Система регистрирует инцидент, формирует уведомление для ИТ‑подразделения, помечает соответствующие проверки как неуспешные.
